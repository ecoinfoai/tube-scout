# Tube Scout v3.1 — 보고서 필터링 및 PDF 종합 출력

## Background

부산보건대 간호학과 YouTube 채널에 214개 강의 영상이 있으며, Tube Scout v1~v3으로 수집/분석/보고서 생성 파이프라인이 구축되었다. 현재 개별 영상 HTML 보고서 214개와 채널 종합 보고서 1개를 생성할 수 있다.

그러나 실무에서는 "홍길동 교수의 2025년 2학기 감염미생물학 영상만 골라서 하나의 보고서로 제출"하는 상황이 발생한다. 현재는 214개 전체에 대해 일괄 생성만 가능하고, 개별 HTML 파일을 하나씩 열어봐야 한다.

## Problem

- **학과 관리자(DX지원센터장)**가 특정 교수/교과목/기간의 영상만 추출하여 분석하고 싶으나, 현재는 전체 영상에 대해서만 보고서를 생성할 수 있다
- 학과장이나 교무과에 보고할 때 **개별 HTML 파일 214개는 실용성이 없다** — 하나의 문서로 묶어서 인쇄하거나 이메일로 보내야 한다
- 보고서에 **표지, 목차, 페이지 번호가 없어** 공식 문서로 활용하기 어렵다

## Desired Outcome

관리자가 조건(교수명, 교과목, 기간 등)을 지정하면 해당 영상만 선별되고, 표지·목차·페이지번호가 포함된 하나의 PDF 종합 보고서로 즉시 출력할 수 있다.

### User Scenarios

- **학과 관리자**는 특정 교수의 특정 학기 영상만 골라 분석 보고서를 생성하고 싶다. 왜냐하면 학과장에게 교수별 강의 영상 현황을 보고해야 하기 때문이다.
- **학과 관리자**는 여러 영상의 분석 결과를 하나의 PDF로 묶어 출력하고 싶다. 왜냐하면 교무과에 제출할 때 단일 문서가 필요하기 때문이다.
- **학과 관리자**는 보고서 생성 전에 필터 결과를 미리 확인하고 싶다. 왜냐하면 214개 중 의도한 영상만 포함되었는지 검증한 뒤 출력해야 하기 때문이다.

## Constraints

- 키워드/기간 필터는 기존 수집 데이터(videos_meta.json)만으로 동작해야 한다 — 제목 파싱(parsed_titles) 없이도 기본 필터링이 가능해야 함
- 영상이 전부 비공개/일부공개 상태이므로 댓글 데이터는 없다 — 댓글 기반 섹션은 데이터 없을 때 우아하게 생략되어야 함
- PDF 출력물은 인쇄 시 가독성을 유지해야 한다 — 차트/테이블이 페이지 경��에서 잘리지 않아야 함

## Success Criteria

- SC-1: 키워드 필터 — "감염미생물학"으로 필터 시 해당 교과목 영상만 정확히 추출된다
- SC-2: 기간 필터 — 게시일 범위 지정 시 범위 내 영상만 포함된다
- SC-3: PDF 종합 출력 — 표지, 자동 생성 목차, 페이지 번호("p. N / Total")가 포함된 단일 PDF가 생성된다
- SC-4: 영상별 페이지 구분 — 각 영상 보고서가 새 페이지에서 시작된다
- SC-5: 필터 미리보기 — 보고서 생성 전 대상 영상 목록을 확인하고 진행 여부를 선택할 수 있다
- SC-6: 정렬 — 게시일순, 교과목→주차순, 조회수순 중 선택할 수 있다

## Discussion Notes

> 이 기능은 v3(멀티채널 관리)에서 한 발 더 나가되, v4(콘텐츠 품질 분석)와는 다른 축이다. v1 보고서의 실무 활용도를 높이는 작업으로, idea3.1로 번호를 부여했다.
>
> 현재 채널에는 214개 영상이 있고 전부 "홍길동" 교수 영상이다. 비공개 149개, 일부공개 65개로 공개 영상은 없다. 댓글은 0개이며, 자막은 일부공개 영상 43개만 접근 가능하다. 시청유지율은 210개 수집 완료, Analytics 5종(일별지표, 구독자, 트래픽, 디바이스, 재생위치) 수집 완료. 인구통계/지역 데이터는 채널 규모 제한으로 YouTube API가 빈 결과를 반환한다.
>
> 자막 분석(EQS/세그먼트)은 LLM API 키(ANTHROPIC_API_KEY 또는 OPENAI_API_KEY)가 필요하여 현재 미실행 상태이다.
>
> retention 수집 시 API 응답의 row index가 코드와 불일치하는 버그가 발견되어 수정했다 (row[2,3,4] → row[0,1,2]).
