# Tube Scout v3 — 교무과 멀티채널 관리 및 학과 보고서

## 배경

대학 교무과가 전체 학과의 강의 영상 업로드 현황을 점검하는 시나리오. 각 학과별로 YouTube 채널이 운영되며, 강의 영상 제목에 교수명, 교과목, 주차, 차시 정보가 포함된다. 교무과는 모든 학과의 영상 생성 상태를 일괄 점검하고, 교수별 영상 제작 현황을 보고서로 생성해야 한다.

## 핵심 목적

**교무과 관점의 학과 강의 영상 관리 자동화** — 멀티채널 인증, 제목 기반 구조화 검색, 학과/교수별 보고서, 제목 오류 탐지.

## 주요 기능

### 1. 멀티채널 토큰 관리

여러 학과 채널의 OAuth 토큰을 `~/.config/tube-scout/tokens/` 아래에 채널별로 저장하여, 매 실행 시 재인증 없이 자동 접근한다.

```
~/.config/tube-scout/
├── client_secret_*.json           ← OAuth 앱 (1개, 교무과 소유)
└── tokens/
    ├── 간호학과.json              ← 최초 1회 인증 후 자동 갱신
    ├── 물리치료과.json
    ├── 치위생과.json
    └── ...
```

- `tube-scout auth --channel <채널별칭>` — 최초 인증 (브라우저 OAuth 플로우)
- `tube-scout auth --list` — 인증된 채널 목록 표시
- `tube-scout auth --revoke <채널별칭>` — 토큰 삭제
- 토큰은 refresh_token 기반 자동 갱신 (비밀번호 변경/권한 해제 시에만 재인증)
- 파일 기반 저장으로 차후 agenix 암호화 전환 용이 (경로만 변경하면 됨)
- 환경변수 `TUBE_SCOUT_TOKENS_DIR`로 토큰 디렉터리 오버라이드 가능 (agenix 복호화 경로 지정용)

### 2. 학과 강의 영상 분석 보고서

학과 채널의 전체 영상을 분석하여 교무과 보고용 문서를 생성한다.

#### 2.1 학과 전체 개요

| 메트릭 | 설명 |
|--------|------|
| 총 영상 수 | 채널 내 전체 영상 |
| 교수별 영상 수 | 제목에서 교수명 파싱 |
| 교과목별 영상 수 | 제목에서 교과목명 파싱 |
| 학기별 영상 수 | 제목 학년도 + 업로드 시점으로 학기 판별 |
| 총 재생시간 | 전체 영상 합산 |
| 총 조회수 | 전체 영상 합산 |

#### 2.2 교수별 상세 분석

| 메트릭 | 설명 |
|--------|------|
| 영상 수 | 해당 교수의 전체 영상 |
| 담당 교과목 목록 | 제목에서 추출 |
| 교과목별 영상 수 | 교과목 × 학기 매트릭스 |
| 주차 커버리지 | 1~16주 중 업로드한 주차 비율 |
| 차시 완성도 | 주차당 평균 차시 수 |
| 총/평균 재생시간 | 영상 길이 분석 |
| 총/평균 조회수 | 시청 현황 |
| 영상 길이 추이 | 학기별 평균 길이 변화 |
| 업로드 타이밍 | 주차 시작일 대비 업로드 지연일 |

#### 2.3 준수율(Compliance) 분석

| 메트릭 | 설명 |
|--------|------|
| 주차별 업로드 현황 | 히트맵 (교수 × 주차) |
| 누락 주차 목록 | 업로드되지 않은 주차 |
| 업로드 마감 준수율 | 해당 주차 시작일 이전 업로드 비율 |
| 미업로드 교수 목록 | 학기 중 영상 0개인 교수 |

#### 2.4 시청 현황 (Analytics API 접근 시)

| 메트릭 | 설명 |
|--------|------|
| 교수별 총 조회수 / 시청 시간 | 채널 소유자 인증 필요 |
| 교과목별 평균 시청 유지율 | Retention 데이터 |
| 학생 접근 빈도 | 일별 조회수 패턴 |

#### 2.5 보고서 출력 형식

- **HTML**: 인터랙티브 차트 포함 (plotly), 브라우저에서 열람
- **Excel (xlsx)**: 교무과 보고용 — 시트별로 학과 개요, 교수별 상세, 준수율, 이상 탐지
- **PDF**: 인쇄용 요약 보고서

### 3. 구조화 검색 조건 (search_clips.yaml)

강의 영상 제목에 포함된 구조 정보(교수명, 교과목, 주차, 차시)를 파싱하여 구조화된 데이터로 변환하고, YAML 파일로 검색 조건을 정의한다.

#### 3.1 제목 파싱 규칙

대학 강의 영상 제목의 일반적 패턴:

```
홍길동 2026 간호학과 인체구조와기능 4주차 2차시
감염미생물학 2024 2학기 홍길동 2주차 1차시
24-1/25-1 홍길동/김영희 융합헬스케어4.0 3주차 1차시 (간호학과)
2023학년도 2학기 국어 5주차 (홍길동)
```

파싱 결과 구조:

| 필드 | 타입 | 예시 |
|------|------|------|
| professor | str | "홍길동" |
| course | str | "인체구조와기능" |
| year | int | 2026 |
| semester | int \| None | 1 또는 2 (추정 가능 시) |
| week | int | 4 |
| session | int | 2 |
| department | str \| None | "간호학과" |

- 파싱 규칙은 정규식 기반, 설정 파일로 커스터마이즈 가능
- 파싱 실패 시 원본 제목 유지 + `parse_error` 플래그

#### 3.2 search_clips.yaml 형식

```yaml
# 검색 조건 정의
filters:
  professor: "홍길동"          # 교수명 (부분 일치)
  course: "인체구조와기능"      # 교과목명 (부분 일치)
  year: 2026                   # 학년도
  semester: 1                  # 학기 (1 또는 2)
  week_range: [1, 8]           # 주차 범위
  session: null                # null = 전체 차시

# 복수 조건 (OR)
queries:
  - professor: "홍길동"
    course: "감염미생물학"
    year: 2024
  - professor: "김영희"
    course: "융합헬스케어"
    year: 2025

# 제외 조건
exclude:
  title_contains: ["핵심영상", "보완영상", "질문응답"]
```

- `tube-scout search --config search_clips.yaml` — YAML 기반 검색
- `tube-scout search --professor 홍길동 --year 2026` — CLI 직접 입력
- 결과: 필터링된 영상 목록 + 파싱된 구조 데이터 (JSON/테이블)

### 4. 제목 오류 및 이상 탐지

제목 파싱 결과와 영상 메타데이터를 교차 검증하여 잠재적 오류를 탐지한다.

#### 4.1 탐지 규칙

| 규칙 ID | 탐지 항목 | 판단 기준 | 심각도 |
|---------|----------|----------|--------|
| V-001 | 학년도 불일치 | 제목 학년도와 업로드 연도 차이 > 1년 | WARNING |
| V-002 | 중복 영상 | 같은 교수+교과목+주차+차시 조합 2개 이상 | ERROR |
| V-003 | 주차 번호 이상 | 주차 > 16 또는 주차 ≤ 0 | ERROR |
| V-004 | 교수명 표기 불일치 | 동일 교수 추정 but 표기 다름 (편집거리 ≤ 2) | WARNING |
| V-005 | 제목 패턴 이탈 | 파싱 실패 또는 필수 필드 누락 | WARNING |
| V-006 | 차시 연속성 결여 | 1차시 없이 2차시만 존재, 또는 역순 | WARNING |
| V-007 | 영상 길이 이상치 | 같은 교과목 평균 대비 ±3σ 벗어남 (< 1분 또는 > 120분) | INFO |
| V-008 | 누락 주차 | 중간 주차 빠짐 (1,2,4,5 → 3주차 누락) | WARNING |
| V-009 | 학기 중 미업로드 기간 | 연속 2주 이상 업로드 없음 | INFO |

#### 4.2 탐지 한계 인지

- **영상 재사용 탐지**: 콘텐츠 기반 재사용 탐지(자막 해시, 유사도 비교)는 idea4(콘텐츠 품질 관리) 범위. idea3에서는 제목/메타데이터 기반 이상 탐지만 수행
- **제목 규칙 미준수**: 제목에 필수 정보가 누락된 경우 파싱 자체가 불가 → V-005로 보고하고, 교무과의 제목 규칙 교육 필요성 제시
- **보완 영상**: "보완영상", "핵심영상", "질문응답" 등 정규 차시가 아닌 보조 영상은 별도 카테고리로 분류
- **공동 강의**: "홍길동/김영희" 같은 복수 교수 영상은 양쪽 교수에 모두 카운트

#### 4.3 출력

- 이상 탐지 결과는 보고서의 별도 섹션에 표시
- 심각도별 색상 구분 (ERROR: 빨강, WARNING: 노랑, INFO: 파랑)
- 교수별 이상 건수 요약
- Excel 보고서에는 "이상 탐지" 시트 별도 생성

## 데이터 저장

수집·파싱·분석된 모든 데이터는 보고서 생성과 별도로 구조화하여 저장한다. idea4(콘텐츠 품질 관리)에서도 이 데이터를 참조한다.

모든 출력은 `./output/report-YYYYMMDD-HHMM/` 아래에 실행 시점 기준으로 저장하여, 이전 실행 결과를 보존하고 연도별·학기별 비교 시 재추출 없이 참조 가능하게 한다.

```
output/
└── report-20260404-1211/               ← 실행 시점
    ├── raw/
    │   └── channels/{channel_id}/
    │       ├── videos_meta.json        ← 전체 영상 메타데이터
    │       └── channel_meta.json       ← 채널 정보
    ├── parsed/
    │   └── {channel_id}/
    │       ├── parsed_titles.json      ← 제목 파싱 결과 (교수/교과목/주차/차시)
    │       ├── professors.json         ← 교수 목록 + 교과목 매핑
    │       └── courses.json            ← 교과목 목록 + 영상 매핑
    ├── validation/
    │   └── {channel_id}/
    │       └── {year}_{semester}.json  ← 이상 탐지 결과 (V-001~V-009)
    └── reports/
        └── department/
            └── {channel_id}_{year}_{semester}.{html|xlsx|pdf}
```

- 각 실행은 독립 디렉터리로 격리 (이전 결과 보존)
- `output/latest` 심볼릭 링크로 최신 결과 접근
- `--output-dir` 옵션으로 오버라이드 가능
- idea4에서 이전 실행의 자막/fingerprint를 참조하여 연도별 diff 비교

## 데이터 소스

- **YouTube Data API v3** — 영상 메타데이터 (기존)
- **YouTube Analytics API** — 시청 데이터 (기존, 채널 소유자 인증 필요)
- **search_clips.yaml** — 검색 조건 정의 (신규)
- **학사 일정** — academic_calendar.json (기존, 준수율 분석에 활용)

## CLI 명령 (신규/확장)

```
tube-scout auth --channel <별칭>       # 채널 최초 인증
tube-scout auth --list                  # 인증된 채널 목록
tube-scout auth --revoke <별칭>        # 토큰 삭제

tube-scout search --config search_clips.yaml   # YAML 기반 검색
tube-scout search --professor 홍길동 --year 2026  # CLI 직접 검색

tube-scout report department --channel <별칭>   # 학과 보고서 생성
  --format html|xlsx|pdf
  --output-dir ./reports/
  --year 2026
  --semester 1

tube-scout validate --channel <별칭>    # 제목 오류 탐지
  --year 2026
  --semester 1
  --output report|json|table
```

## 워크플로우

```
1. 교무과 담당자: 각 학과별 tube-scout auth --channel <학과명> 실행 (최초 1회)
2. tube-scout search --config search_clips.yaml   # 분석 대상 확인
3. tube-scout collect videos --channel <학과명>    # 영상 메타데이터 수집
4. tube-scout validate --channel <학과명>          # 제목 오류 탐지
5. tube-scout report department --channel <학과명> --format xlsx  # 보고서 생성
6. 필요 시: tube-scout collect analytics --channel <학과명>  # 시청 데이터 수집
```

## 기대 효과

- 교무과에서 전체 학과의 강의 영상 업로드 현황을 **자동 점검**
- 교수별 영상 제작 현황을 **정량적으로 분석**하여 성과 관리
- 제목 오류를 **사전 탐지**하여 관리 품질 향상
- 멀티채널 인증으로 **운영 편의성** 확보
- 보고서 자동 생성으로 **보고 업무 시간 절감**
