## Clarifications

### Session 2026-04-01

- Q: 수집 데이터의 영속성 전략은? → A: 로컬 파일 기반 (JSON 메타데이터 + Parquet 시계열 데이터). data/ 디렉토리에 저장, 별도 DB 서버 불필요.
- Q: 사용자 인터페이스 형태는? → A: CLI 기본 + Jupyter 노트북 리포트 내보내기 지원. 자동화(스크립트/cron)와 대화형 탐색 모두 가능.

- Q: YouTube API 할당량 초과 시 동작은? → A: 진행 상황을 저장 후 중단, 할당량 복구 시 이어서 수집(Resume). 이미 수집된 데이터는 보존.
- Q: 데이터 갱신 주기는? → A: 수동 실행 전용 (사용자가 CLI 명령으로 직접 트리거). 자동화 필요 시 외부 스케줄러(cron 등)와 조합.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 교수 영상 식별 및 기본 메트릭 수집 (Priority: P1)

교수(채널 운영자)가 자신의 YouTube 채널 ID와 이름을 입력하면, 시스템이 해당 채널에서 제목에 교수명이 포함된 모든 영상을 자동으로 식별하고, 각 영상의 기본 메트릭(조회수, 시청 완료율, 평균 시청 시간, 좋아요/싫어요 비율)을 수집하여 대시보드 형태로 보여준다.

**Why this priority**: 모든 후속 분석의 기반이 되는 데이터 수집 단계. 이것 없이는 어떤 분석도 불가능하다.

**Independent Test**: 채널 ID와 교수명을 입력하고, 올바른 영상 목록이 반환되며 각 영상의 기본 메트릭이 표시되는지 확인하여 독립적으로 테스트 가능. 이 단계만으로도 교수는 자신의 영상 현황을 한눈에 파악할 수 있다.

**Acceptance Scenarios**:

1. **Given** 유효한 채널 ID와 교수명이 입력됨, **When** 영상 수집을 실행, **Then** 제목에 교수명이 포함된 모든 영상 목록이 반환되고 각 영상의 조회수, 좋아요 수, 댓글 수, 영상 길이가 표시됨
2. **Given** 채널에 교수명이 포함된 영상이 없음, **When** 영상 수집을 실행, **Then** 결과 없음 메시지와 함께 검색 조건 확인을 안내함
3. **Given** 이전에 수집한 영상 목록이 존재함, **When** 신규 영상이 채널에 업로드된 후 수집을 재실행, **Then** 신규 영상이 목록에 추가됨

---

### User Story 2 - 시청 패턴 심층 분석 (Priority: P1)

교수가 특정 영상을 선택하면, 구간별 시청 유지율 곡선, 되감기 빈도 구간(Rewatch Hotspot), 건너뛰기 구간(Skip Zone)을 시각화하여 학생들이 어려워하거나 흥미를 잃는 구간을 정확히 파악할 수 있다.

**Why this priority**: 이 기능이 프로젝트의 핵심 가치 — "학생이 어려워하는 구간"과 "최적 영상 구성"에 대한 직접적 인사이트를 제공한다.

**Independent Test**: 특정 영상을 선택했을 때 시청 유지율 곡선이 올바르게 렌더링되고, Rewatch Hotspot과 Skip Zone이 하이라이트되는지 확인. 교수는 이 정보만으로도 강의 영상 개선 방향을 잡을 수 있다.

**Acceptance Scenarios**:

1. **Given** Analytics API 접근 권한이 있는 영상이 선택됨, **When** 시청 패턴 분석을 요청, **Then** 시간축 기반 시청 유지율 곡선이 표시되고 Rewatch Hotspot과 Skip Zone이 색상으로 구분되어 하이라이트됨
2. **Given** 시청 데이터가 충분한 영상(최소 100회 이상 조회), **When** Rewatch Hotspot 분석, **Then** 되감기 빈도가 평균 대비 유의미하게 높은 구간이 자동으로 식별되고 해당 구간의 시작/종료 시간이 표시됨
3. **Given** Analytics API 접근 권한이 없음, **When** 시청 패턴 분석을 요청, **Then** 권한 부족 안내와 함께 기본 메트릭(Data API 기반)만 표시됨

---

### User Story 3 - 댓글 하이브리드 분석 (Priority: P2)

교수가 특정 영상 또는 전체 영상의 댓글을 분석하면, 감성(긍정/부정/중립), 논의 주제(Topic), 학생 질문이 자동 분류되어 표시된다. 추출된 질문은 되감기 구간과 교차 분석되어 난이도 구간을 이중 검증한다.

**Why this priority**: 시청 패턴 데이터와 결합하여 "왜 어려운지"에 대한 질적 인사이트를 제공한다. 시청 데이터만으로는 이유를 알 수 없으므로, 댓글 분석이 이를 보완한다.

**Independent Test**: 영상 댓글을 수집하고 감성/토픽/질문으로 분류된 결과가 표시되는지 확인. 이 기능만으로도 교수는 학생 피드백의 전체 그림을 파악할 수 있다.

**Acceptance Scenarios**:

1. **Given** 댓글이 10개 이상인 영상이 선택됨, **When** 댓글 분석을 실행, **Then** 각 댓글이 감성(긍정/부정/중립), 주요 토픽, 질문 여부로 분류되어 요약이 표시됨
2. **Given** 댓글 분석 결과와 시청 패턴 데이터가 모두 있음, **When** 교차 분석을 요청, **Then** 학생 질문이 집중된 주제와 Rewatch Hotspot의 시간 구간이 대응되어 표시됨
3. **Given** 댓글이 없거나 비활성화된 영상, **When** 댓글 분석을 실행, **Then** 댓글 데이터 없음 안내와 함께 다른 분석 수단 제안

---

### User Story 4 - LLM 자막 분석 및 난이도 예측 (Priority: P2)

교수가 영상의 자막(Transcript)을 분석하면, 시스템이 자동으로 챕터를 분할하고, 각 구간의 핵심 내용을 요약하며, 어휘/개념 밀도를 기반으로 난이도를 사전 예측한다. 주제별 자동 태깅도 수행된다.

**Why this priority**: 시청 데이터가 아직 축적되지 않은 신규 영상에도 적용 가능하며, 콘텐츠 자체의 구조적 문제를 진단할 수 있다.

**Independent Test**: 자막이 있는 영상을 선택하고 챕터 분할, 요약, 난이도 점수가 생성되는지 확인. 신규 업로드 직후에도 즉시 활용 가능하다.

**Acceptance Scenarios**:

1. **Given** 자막이 존재하는 영상이 선택됨, **When** 자막 분석을 실행, **Then** 영상이 의미론적 단위로 분절되고 각 구간에 제목, 요약, 난이도 점수가 부여됨
2. **Given** 자막이 존재하지 않는 영상, **When** 자막 분석을 실행, **Then** 자동 음성 인식(STT)을 통해 자막을 생성한 후 분석을 수행하거나, 자막 생성 불가 시 해당 안내를 표시
3. **Given** 시청 패턴 데이터와 자막 난이도 예측이 모두 있음, **When** 비교를 요청, **Then** 예측 난이도와 실제 Rewatch Hotspot의 일치도가 표시됨

---

### User Story 5 - 분석 리포트 생성 (Priority: P2)

교수가 리포트 생성을 요청하면, 개별 영상 리포트(성과 요약 + 난이도 구간 하이라이트 + EQS 점수)와 채널 종합 리포트(트렌드, 비교 분석, 시계열 예측)가 생성된다. 리포트에는 데이터 기반 개선 제안이 포함된다.

**Why this priority**: 수집된 데이터와 분석 결과를 교수가 실제로 활용할 수 있는 형태로 제공하는 최종 산출물이다.

**Independent Test**: 분석이 완료된 영상에 대해 리포트를 생성하고, 필수 섹션(성과 요약, 난이도 구간, 개선 제안)이 포함되었는지 확인.

**Acceptance Scenarios**:

1. **Given** 기본 메트릭과 시청 패턴 분석이 완료된 영상, **When** 영상별 리포트 생성을 요청, **Then** 성과 요약, 난이도 구간 하이라이트, 시청 유지율 곡선, 개선 제안이 포함된 리포트가 생성됨
2. **Given** 10개 이상의 영상이 분석 완료됨, **When** 채널 종합 리포트를 요청, **Then** 영상 간 비교 분석, 시계열 트렌드, 주제별 성과 분석이 포함된 종합 리포트가 생성됨
3. **Given** 리포트가 생성됨, **When** 리포트를 내보내기, **Then** 지정된 형식(HTML 또는 Jupyter Notebook)으로 파일이 저장됨

---

### User Story 6 - 교육 품질 자동 스코어링 (Priority: P3)

교수가 자신의 영상에 대해 교육 품질 점수(EQS)를 확인하면, RACED 5축(관련성, 정확성, 명료성, 참여도, 깊이) 기준의 평가 결과가 표시된다. 조회수와 독립적인 교육적 가치 판단을 제공한다.

**Why this priority**: 기본 메트릭과 시청 패턴 분석이 선행되어야 의미 있는 품질 점수를 산출할 수 있으며, 이 기능은 기존 분석 위에 추가 가치를 더하는 고도화 기능이다.

**Independent Test**: 분석 완료된 영상에 대해 EQS 점수가 5축별로 표시되고, 각 축에 대한 설명이 제공되는지 확인.

**Acceptance Scenarios**:

1. **Given** 자막 분석과 시청 패턴 분석이 완료된 영상, **When** EQS 평가를 요청, **Then** RACED 5축 각각의 점수와 종합 점수가 표시됨
2. **Given** 복수의 영상에 EQS가 산출됨, **When** 비교를 요청, **Then** 영상 간 EQS 비교 차트와 개선이 필요한 축이 하이라이트됨

---

### User Story 7 - 시계열 예측 및 이상 탐지 (Priority: P3)

교수가 채널 성과 예측을 요청하면, 향후 조회수/시청 시간 트렌드가 예측되고, 과거 데이터에서 이상치(시험 기간 급증, 방학 급감 등)가 자동으로 식별되어 학기 주기 패턴이 시각화된다.

**Why this priority**: 충분한 과거 데이터가 축적된 후에 의미 있는 예측이 가능하므로, 다른 기능이 먼저 운영되어야 한다.

**Independent Test**: 6개월 이상의 시청 데이터가 있는 채널에서 예측 차트가 생성되고 이상치 구간이 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 6개월 이상의 시청 데이터가 존재, **When** 시계열 예측을 요청, **Then** 향후 시청 트렌드 예측 그래프와 신뢰 구간이 표시됨
2. **Given** 시계열 분석이 수행됨, **When** 이상치 탐지를 요청, **Then** 급격한 시청 패턴 변화가 있는 구간이 하이라이트되고 해당 시점의 가능한 원인(학사 일정 등)이 제안됨

---

### Edge Cases

- 채널에 교수명과 유사하지만 정확히 일치하지 않는 영상 제목이 있을 때 어떻게 처리하는가? (예: "홍길동" vs "홍길동 교수" vs "홍길동교수")
- YouTube API 할당량(Quota)이 초과되었을 때: 진행 상황을 로컬 파일에 저장하고 중단한다. 할당량 복구 후 재실행 시 중단 지점부터 이어서 수집한다(Resume). 이미 수집된 데이터는 보존된다.
- 자막이 자동 생성된 경우(정확도가 낮을 수 있음)와 수동 자막의 품질 차이를 어떻게 반영하는가?
- 비공개/일부 공개 영상에 대한 접근 권한 처리
- 한국어와 영어가 혼용된 댓글의 감성 분석 정확도
- 장기간(1년 이상) 데이터 수집 시 API 응답 변경이나 영상 삭제에 대한 처리

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 YouTube 채널 ID를 입력받아 해당 채널의 영상 목록을 조회할 수 있어야 한다. v1은 단일 채널/단일 교수를 지원하며, 설정 파일 구조는 복수 채널/교수 확장이 가능한 리스트 형태로 설계한다
- **FR-002**: 시스템은 영상 제목에서 지정된 교수명을 기반으로 대상 영상을 필터링할 수 있어야 한다. 부분 일치(교수명이 제목의 일부로 포함)를 지원해야 한다
- **FR-003**: 시스템은 필터링된 각 영상의 기본 메트릭을 수집해야 한다. Data API 기본 메트릭(조회수, 좋아요 수, 댓글 수, 영상 길이)은 US1에서, Analytics API 의존 메트릭(고유 시청자 수, 시청 완료율, 평균 시청 시간)은 US2에서 수집한다. YouTube API 정책 변경(2021.11)으로 싫어요 수는 수집 불가하다
- **FR-004**: 시스템은 구간별 시청 유지율 곡선을 시각화하고, 되감기 빈도가 높은 구간(Rewatch Hotspot)과 건너뛰기가 많은 구간(Skip Zone)을 자동으로 식별해야 한다
- **FR-005**: 시스템은 영상 댓글을 수집하고 감성(긍정/부정/중립), 토픽, 질문 여부를 분류할 수 있어야 한다
- **FR-006**: 시스템은 영상 자막(Transcript)을 추출하고, 의미론적 단위로 챕터를 자동 분할하며 각 구간의 요약과 난이도를 산출할 수 있어야 한다
- **FR-007**: 시스템은 자막이 없는 영상에 대해 선택적으로 음성 인식(STT)을 통한 자막 자동 생성을 지원해야 한다. Whisper가 설치된 환경에서만 STT를 시도하며, 미설치 시 해당 영상을 skip하고 로그에 기록한다
- **FR-008**: 시스템은 RACED 5축(관련성, 정확성, 명료성, 참여도, 깊이) 기반 교육 품질 점수(EQS)를 산출할 수 있어야 한다
- **FR-009**: 시스템은 과거 시청 데이터를 기반으로 향후 조회수/시청 시간을 예측하고, 이상치를 자동 탐지할 수 있어야 한다
- **FR-010**: 시스템은 영상별 개별 리포트와 채널 종합 리포트를 생성할 수 있어야 한다. 리포트에는 데이터 기반 개선 제안이 포함되어야 한다
- **FR-011**: 시스템은 댓글에서 추출된 학생 질문과 Rewatch Hotspot을 교차 분석하여 난이도 구간을 이중 검증할 수 있어야 한다
- **FR-012**: 시스템은 이전에 수집한 데이터를 로컬 파일(JSON 메타데이터 + Parquet 시계열 데이터)로 data/ 디렉토리에 보존하고, 사용자가 수집 명령을 실행할 때 신규 업로드된 영상을 감지하여 목록에 추가할 수 있어야 한다. 자동 주기 실행은 외부 스케줄러(cron 등)에 위임한다
- **FR-013**: 시스템은 모든 인증 정보를 환경변수를 통해서만 참조해야 하며, 코드 내에 인증 경로나 키값을 하드코딩하지 않아야 한다
- **FR-014**: 시스템은 리포트를 파일(HTML 등)로 내보낼 수 있어야 한다
- **FR-015**: 시스템은 YouTube API 할당량 초과 시 수집 진행 상황을 저장하고 안전하게 중단해야 하며, 재실행 시 중단 지점부터 이어서 수집(Resume)할 수 있어야 한다

### Key Entities

- **Channel**: YouTube 채널. 채널 ID, 채널명, 소유자 정보를 포함. 분석의 최상위 단위
- **Video**: 채널 내 개별 영상. 제목, 업로드일, 영상 길이, 기본 메트릭을 포함. 교수명 필터링의 대상
- **Viewing Pattern**: 영상의 구간별 시청 데이터. 시청 유지율 곡선, Rewatch Hotspot, Skip Zone 정보를 포함
- **Comment**: 영상에 달린 댓글. 원문, 감성 분류, 토픽, 질문 여부를 포함
- **Transcript Segment**: 자막 기반으로 분절된 영상 구간. 시작/종료 시간, 텍스트, 요약, 난이도 점수를 포함
- **Quality Score (EQS)**: 영상의 교육 품질 평가. RACED 5축 개별 점수와 종합 점수를 포함
- **Report**: 분석 결과물. 영상별 리포트, 채널 종합 리포트, 댓글 인사이트 리포트의 유형이 있음
- **Forecast**: 시계열 예측 결과. 예측값, 신뢰 구간, 이상치 정보를 포함

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 교수가 채널 ID와 이름을 입력한 후 5분 이내에 전체 대상 영상 목록과 기본 메트릭을 확인할 수 있다
- **SC-002**: 영상의 난이도 구간(Rewatch Hotspot) 식별 정확도가 수동 분석 결과 대비 80% 이상 일치한다
- **SC-003**: 댓글 감성 분류 정확도가 90% 이상이다 (한국어 댓글 기준)
- **SC-004**: 자막 기반 자동 챕터 분할이 영상 내용의 주제 전환점을 80% 이상 정확히 식별한다
- **SC-005**: 리포트 생성 후, 교수가 최소 1가지 이상의 구체적인 영상 개선 행동을 식별할 수 있다 (사용자 테스트 기준)
- **SC-006**: 시계열 예측 모델의 평균 절대 오차(MAE)가 15% 이내이다 (6개월 이상 데이터 기준)
- **SC-007**: 시스템이 100개 이상의 영상을 포함하는 채널을 안정적으로 분석할 수 있다

## Assumptions

- 분석 대상 YouTube 채널의 소유자 또는 관리자로부터 OAuth 인증 협조를 받을 수 있다 (YouTube Analytics API 접근에 필수)
- 대부분의 강의 영상에 한국어 자막(자동 생성 포함)이 존재한다
- 교수명은 영상 제목에 일관된 형태로 포함되어 있다 (완전히 다른 표기가 사용되는 경우는 범위 외)
- YouTube API 할당량 내에서 분석이 가능한 규모의 채널이다 (일일 10,000 단위 기준)
- 사용자는 명령줄(CLI)을 주된 인터페이스로 사용하며, Jupyter 노트북을 통한 리포트 확인 및 대화형 탐색도 지원한다 (별도 웹 UI는 v1 범위 외)
- 인증 정보는 agenix로 암호화되어 중앙 관리되며, flake.nix devShell을 통해 환경변수로 주입된다
- LLM API(GPT-4o 등) 호출 비용은 분석 규모에 비례하며, 사용자가 이를 수용한다
