# Feature Specification: 보고서 필터링 및 PDF 종합 출력

**Feature Branch**: `004-report-filter-pdf-bundle`
**Created**: 2026-04-04
**Status**: Draft
**Input**: User description: "idea/idea3.1.md"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 키워드/기간으로 영상 필터링 후 보고서 생성 (Priority: P1)

학과 관리자가 특정 교과목명이나 교수명을 키워드로 지정하거나, 게시일 범위를 지정하여 해당 영상만 선별한 뒤 보고서를 생성한다. 214개 전체가 아닌, 관심 대상 영상에 대해서만 분석 결과를 확인할 수 있다.

**Why this priority**: 필터링 없이는 PDF 종합 출력도 의미가 없다. 대상 영상을 정확히 선별하는 것이 모든 후속 기능의 전제 조건이다.

**Independent Test**: 키워드 "감염미생물학"으로 필터 실행 시 해당 교과목 영상만 보고서로 생성되는지 확인. 기간 필터 "2026-01-01 ~ 2026-04-01" 지정 시 범위 밖 영상이 제외되는지 확인.

**Acceptance Scenarios**:

1. **Given** 214개 영상 메타데이터가 수집된 상태, **When** 키워드 "인체구조와기능"으로 보고서 생성을 요청, **Then** 제목에 "인체구조와기능"이 포함된 영상만 보고서에 포함된다
2. **Given** 214개 영상 메타데이터가 수집된 상태, **When** 게시일 범위 2026-01-01~2026-04-01로 필터, **Then** 해당 기간에 게시된 영상만 포함된다
3. **Given** 키워드와 기간 필터를 동시에 지정, **When** 보고서 생성, **Then** 두 조건을 모두 만족하는 영상만 포함된다 (AND 조건)
4. **Given** 필터 조건에 맞는 영상이 0개, **When** 보고서 생성 요청, **Then** 해당 조건에 맞는 영상이 없다는 안내를 표시하고 빈 보고서를 생성하지 않는다

---

### User Story 2 - PDF 종합 보고서 출력 (Priority: P1)

학과 관리자가 필터링된 영상의 분석 결과를 표지, 목차, 페이지 번호가 포함된 하나의 PDF 문서로 출력한다. 교무과나 학과장에게 제출할 수 있는 공식 문서 형태이다.

**Why this priority**: 개별 HTML 파일은 실무 보고용으로 활용할 수 없다. 단일 PDF 출력이 이 기능의 핵심 가치이다.

**Independent Test**: 필터링된 10개 영상에 대해 PDF 생성을 실행하고, 출력물에 표지/목차/페이지번호가 포함되어 있는지, 각 영상이 새 페이지에서 시작하는지 확인.

**Acceptance Scenarios**:

1. **Given** 필터로 24개 영상이 선택된 상태, **When** PDF 종합 보고서 생성 요청, **Then** 표지 + 목차 + 24개 영상 상세가 포함된 단일 PDF 파일이 생성된다
2. **Given** PDF 보고서가 생성된 상태, **When** 목차 페이지를 확인, **Then** 각 영상 제목과 해당 페이지 번호가 정확히 표시되어 있다
3. **Given** PDF 보고서가 생성된 상태, **When** 페이지 하단을 확인, **Then** "p. N / Total" 형식의 페이지 번호가 모든 페이지에 표시된다
4. **Given** PDF 보고서가 생성된 상태, **When** 영상별 섹션 경계를 확인, **Then** 각 영상 보고서가 새 페이지에서 시작된다

---

### User Story 3 - 필터 결과 미리보기 (Priority: P2)

학과 관리자가 보고서 생성 전에 필터 조건에 맞는 영상 목록을 먼저 확인하고, 의도한 대로 영상이 선별되었는지 검증한 뒤 진행 여부를 결정한다.

**Why this priority**: 214개 중 정확한 대상만 포함되었는지 확인 없이 PDF를 생성하면 시간 낭비가 발생한다. 다만 필터링과 PDF 생성이 먼저 동작해야 미리보기도 의미가 있다.

**Independent Test**: dry-run 모드로 필터 실행 시 보고서는 생성되지 않고 대상 영상 목록만 표시되는지 확인.

**Acceptance Scenarios**:

1. **Given** 키워드 "감염미생물학"과 dry-run 모드 지정, **When** 보고서 생성 요청, **Then** 대상 영상 목록(제목, 게시일)이 표시되고 실제 보고서는 생성되지 않는다
2. **Given** 미리보기(--dry-run)로 대상 영상 목록을 확인한 상태, **When** 동일 필터 조건으로 --dry-run 없이 재실행, **Then** 확인된 영상들에 대해 보고서가 생성된다

---

### User Story 4 - 영상 정렬 옵션 (Priority: P3)

학과 관리자가 종합 보고서 내 영상 순서를 게시일순, 교과목→주차순, 조회수순 중 선택할 수 있다.

**Why this priority**: 보고서의 가독성을 높이는 부가 기능으로, 핵심 필터링/PDF 출력이 동작한 후에 추가하면 된다.

**Independent Test**: 동일 영상 세트에 대해 정렬 옵션을 바꿔가며 PDF를 생성하고, 영상 순서가 지정한 기준에 따라 달라지는지 확인.

**Acceptance Scenarios**:

1. **Given** 24개 영상이 선택된 상태, **When** 게시일 역순 정렬로 PDF 생성, **Then** 최신 영상이 먼저 나온다
2. **Given** 24개 영상이 선택된 상태, **When** 조회수 내림차순 정렬로 PDF 생성, **Then** 가장 많이 본 영상이 먼저 나온다

---

### User Story 5 - 종합 보고서에 채널 요약 포함 (Priority: P3)

학과 관리자가 종합 보고서(bundle)에 필터 대상 영상의 통계 요약 섹션을 포함하여, 영상별 상세 분석 앞에 전체 맥락을 제공한다.

**Why this priority**: 채널 요약은 있으면 좋지만 없어도 영상별 보고서만으로 보고 목적을 달성할 수 있다.

**Independent Test**: bundle 보고서에 요약 섹션이 포함되어 있고, 대상 영상의 통계(영상 수, 총 재생시간, 평균 조회수)가 정확한지 확인.

**Acceptance Scenarios**:

1. **Given** 필터로 24개 영상이 선택된 상태, **When** bundle 보고서 생성, **Then** 표지 다음에 대상 영상의 통계 요약(영상 수, 총 재생시간, 평균 조회수 등)이 표시된다

---

### User Story 6 - 기존 HTML 보고서를 수거하여 PDF 병합 (Priority: P2)

학과 관리자가 이미 생성된 개별 HTML 보고서 파일들을 지정하여, 데이터 재분석 없이 PDF 종합 보고서로 묶는다. 분석이 오래 걸리는 대량 영상의 경우, HTML을 한 번 생성해두고 필요할 때마다 다른 조합으로 PDF를 만들 수 있다.

**Why this priority**: 데이터→PDF 직접 생성(P1)이 기본이지만, 214개 HTML이 이미 존재하는 상황에서 재분석 없이 빠르게 PDF를 만드는 것은 실무적 가치가 크다.

**Independent Test**: `data/reports/video/` 디렉터리에 있는 기존 HTML 파일 10개를 지정하여 PDF 생성 시, 데이터 디렉터리 접근 없이 PDF가 생성되는지 확인.

**Acceptance Scenarios**:

1. **Given** `data/reports/video/`에 214개 HTML 보고서가 존재하는 상태, **When** HTML 디렉터리를 지정하고 키워드 필터와 함께 bundle 생성 요청, **Then** 키워드에 맞는 HTML 파일만 수거하여 표지/목차가 포함된 PDF가 생성된다
2. **Given** 기존 HTML 파일을 사용하여 PDF 생성, **When** 분석 데이터 디렉터리가 없는 환경에서 실행, **Then** 데이터 재분석 없이 HTML 내용 그대로 PDF에 포함된다
3. **Given** HTML 디렉터리에 일부 영상만 보고서가 있는 상태, **When** 필터 조건에 맞지만 HTML이 없는 영상이 있을 때, **Then** 해당 영상은 건너뛰고 존재하는 HTML만으로 PDF를 생성하며, 누락 목록을 안내한다

---

### Edge Cases

- 필터 결과가 0개일 때 — 빈 PDF를 생성하지 않고 안내 메시지 표시
- 필터 결과가 1개일 때 — 목차 없이 표지 + 단일 영상 보고서로 생성
- 영상에 시청유지율 데이터가 없을 때 (4개 실패분) — 해당 섹션을 "데이터 없음"으로 표시하고 나머지는 정상 출력
- 영상에 자막/세그먼트 분석 데이터가 없을 때 — 해당 섹션 생략
- 키워드에 특수문자(괄호, 따옴표 등)가 포함된 경우 — 정상 동작
- 200개 이상 영상이 필터에 걸릴 때 — PDF 크기 경고 표시 후 진행 여부 확인
- `--from-html` 모드에서 HTML 파일이 깨져 있거나 파싱 불가일 때 — 해당 파일 건너뛰고 경고 표시
- `--from-html` 모드에서 필터에 맞는 HTML이 0개일 때 — 안내 메시지 표시, 빈 PDF 생성 안 함

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 제목 키워드로 영상을 필터링할 수 있어야 한다 — 영상 제목에 지정한 문자열이 포함된 영상만 선택
- **FR-002**: 시스템은 게시일 범위로 영상을 필터링할 수 있어야 한다 — 시작일과 종료일 지정
- **FR-003**: 시스템은 복수 필터를 AND 조건으로 조합할 수 있어야 한다 — 키워드 + 기간 동시 적용
- **FR-004**: 시스템은 영상 ID 목록을 직접 지정하여 보고서를 생성할 수 있어야 한다
- **FR-005**: 시스템은 필터 결과를 미리보기(dry-run)로 표시할 수 있어야 한다 — 보고서 생성 없이 대상 목록만 출력
- **FR-006**: 시스템은 `report bundle` 명령을 통해 필터링된 영상들의 분석 결과를 하나의 PDF 문서로 출력할 수 있어야 한다 — `report video`와 별도 명령
- **FR-007**: PDF 문서에 표지가 포함되어야 한다 — 채널명, 필터 조건, 대상 영상 수, 생성일
- **FR-008**: PDF 문서에 목차가 자동 생성되어야 한다 — 각 영상 제목과 페이지 번호
- **FR-009**: PDF 문서의 모든 페이지에 페이지 번호가 표시되어야 한다 — "p. N / Total" 형식
- **FR-010**: PDF 문서에서 각 영상 보고서는 새 페이지에서 시작해야 한다
- **FR-011**: 시스템은 영상 정렬 기준을 선택할 수 있어야 한다 — 게시일순(기본값), 교과목→주차순, 조회수순
- **FR-012**: 분석 데이터가 없는 영상의 보고서 섹션은 우아하게 생략되어야 한다 — 시청유지율, 자막, 세그먼트 등
- **FR-013**: 필터 결과가 0개일 때 빈 보고서를 생성하지 않고 안내 메시지를 표시해야 한다
- **FR-014**: 종합 보고서(bundle)에 필터 대상 영상의 통계 요약 섹션을 포함할 수 있어야 한다
- **FR-015**: `report bundle`은 기존 HTML 보고서 파일 디렉터리를 입력으로 받아 데이터 재분석 없이 PDF를 생성할 수 있어야 한다 — `--from-html` 옵션
- **FR-016**: `--from-html` 모드에서도 키워드/기간 필터가 동작해야 한다 — HTML 파일명(video_id)과 메타데이터를 매칭하여 필터링

### Key Entities

- **VideoFilter**: 영상 선택 조건 — 키워드, 게시일 범위, 영상 ID 목록 등의 조합으로 대상 영상을 결정
- **BundleReport**: 종합 보고서 — 표지, 목차, 통계 요약, 영상별 상세 분석의 집합체
- **CoverPage**: 보고서 표지 — 채널명, 필터 조건 요약, 영상 수, 생성일 등 보고서 식별 정보

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 키워드 필터 시 해당 키워드를 포함하는 영상만 정확히 추출된다 (오탐/누락 0건)
- **SC-002**: 게시일 범위 필터 시 범위 밖 영상이 포함되지 않는다
- **SC-003**: 24개 영상에 대한 PDF 종합 보고서가 3분 이내에 생성된다
- **SC-004**: PDF 보고서에 표지, 목차, 페이지 번호가 빠짐없이 포함된다
- **SC-005**: PDF를 A4 용지에 인쇄했을 때 차트와 테이블이 페이지 경계에서 잘리지 않는다
- **SC-006**: 분석 데이터가 없는 영상도 에러 없이 보고서에 포함된다 (해당 섹션 생략)
- **SC-007**: 기존 HTML 214개를 키워드 필터로 수거하여 PDF 생성 시, 데이터 재분석 없이 1분 이내에 완료된다

## Clarifications

### Session 2026-04-04

- Q: bundle은 별도 명령인가, 기존 report video 확장인가? → A: `report bundle` 신규 명령으로 분리. `report video`는 필터 옵션만 추가하고 개별 HTML 보고서 생성. `report bundle`이 표지/목차/PDF 종합 출력 전담.
- Q: 키워드 매칭 방식 — 공백/대소문자 처리? → A: 단순 부분 문자열 매칭. 입력한 그대로 제목에 포함되는지 검사. 공백 무시나 퍼지 매칭 없음.
- Q: 기존 HTML 파일을 수거하여 PDF로 묶는 기능이 필요한가? → A: 필요. `--from-html` 옵션으로 기존 HTML 디렉터리를 지정하면 데이터 재분석 없이 PDF 생성. 데이터→PDF 직접 생성(기본)과 HTML→PDF 수거(선택) 두 경로 모두 지원.

## Assumptions

- 영상 메타데이터(videos_meta.json)가 이미 수집된 상태에서 동작한다
- 키워드 필터는 제목의 부분 문자열 매칭으로 동작한다 (정규식이 아닌 단순 포함 검사)
- 기간 필터의 날짜는 ISO 8601 형식(YYYY-MM-DD)을 사용한다
- 기본 정렬 순서는 게시일 역순(최신 먼저)이다
- PDF 출력 시 차트는 정적 이미지로 렌더링된다
- 종합 보고서(bundle)에 통계 요약 섹션을 기본 포함한다
- xlsx 종합 출력은 초기 범위에서 제외하고 후속 확장으로 둔다
- 표지 정보: 채널명, 필터 조건 요약, 대상 영상 수, 생성일
